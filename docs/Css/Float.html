    浮动

    传统三种布局方式：标准流（标签默认排列）、浮动流、定位流

    # 1.浮动特性

    float: left | rigth | none;

    1、浮动元素脱离标准流（脱标），不保留原先的位置

    2、浮动的元素会一行内显示并且元素顶部对齐

    3、浮动的元素会具有行内块元素的特性

    4、浮动的盒子只会影响浮动盒子后面的标准流不会影响前面的标准流.

    5、浮动元素不同，只会压住它下面标准流的盒子，但是不会压住下面标准流盒子里面的文字(图片)

    浮动元素常搭配标准流的父元素使用

    # 2.清除浮动

    为什么要清除浮动？

    父盒子要填充多少内容是不好确定的，因而为了方便通常不设置高度，让子元素的内容撑开父盒子。但是如果子元素设置了浮动这就意味着脱离了文档流，恰好此时父盒子未设置高度，就会影响后面标准流的元素。为了避免这种影响，所以要清除浮动

    父盒子没有高度，子元素浮动影响后面的元素

    额外标签法（不常用）

    父元素overflow：hidden

    <style>
    .box {
      width: 400px;
      background-color: blueviolet;
      margin: 0 auto;
      /* 父元素overflow清除浮动 */
      overflow: hidden;
    }
    .box1, .box2 {
      width: 150px;
      height: 150px;
      float: left;
    }
    .box1 {
      background-color: red;
    }
    .box2 {
      background-color: blue;
    }
    .footer {
      height:150px;
      background-color: chartreuse;
    }
  </style>
<body>
  <div class="box">
    <div class="box1">1</div>
    <div class="box2">2</div>
  </div>
  <div class="footer"></div>
</body>

    父元素添加伪元素

    <style>
.clear:after {
      content: "";
      display: block;
      height: 0;
      clear: both;
      visibility: hidden;
    }
    /* IE6、7专用 */
    .clear {
      *zoom:1 ;
    }
</style>
<body>
  <div class="box clear">
    <div class="box1">1</div>
    <div class="box2">2</div>
  </div>
  <div class="footer"></div>
</body>
